# 人工操作手册

1. 在任务索引中选择第一个 `READY` Task，确认所有前置 Task 为 `ACCEPTED`。
2. 确认工作区干净并运行 `New-ModelTaskPrompt.ps1`；需要 commit 时显式传 `-AllowCommit`。确需保留既有修改时，必须同时传入逐项 `-KnownDirtyPath` 和非“无”的 `-KnownDeviation` 原因。
3. 把生成提示词发给执行模型；核对其 `READY` 报告中的目标、白名单和测试。
4. 正确时发送“开始执行 Task <ID>”；错误时指出字段并要求重新预检，不能让它先修改。
5. 收到 `COMPLETED` 后不要立即开始下一 Task；先保存 commit ID 并运行独立复核提示词。
6. `ACCEPTED` 后由人工更新任务索引状态并选择下一 Task；`REWORK` 交回原 Task；`BLOCKED` 记录决定后重新生成提示词。
7. 任何模型都不得 push、merge、清理用户修改或同时执行两个 Task。
