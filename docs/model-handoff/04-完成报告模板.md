# 完成与阻塞报告模板

## 阶段一：只读预检

第一次收到启动提示词后只能返回以下字段，不得修改文件、安装依赖或提交：

```text
状态：READY 或 BLOCKED
Task ID：
我理解的唯一目标：
明确不做：
前置条件检查：
必须读取的文件：
允许修改：
禁止修改：
计划执行的测试：
停止条件：
发现的冲突：
```

## 执行中阻塞

任何停止条件成立时立即停止，并返回：

```text
状态：BLOCKED
Task ID：
已完成步骤：
阻塞位置：
实际证据：
计划预期：
需要人工决定的问题：
工作区修改：
建议选项：
```

## Task 完成

只有必选验证全部执行且退出码为 0、当前 Task 已产生一个 commit、没有当前 Task 的未提交修改、没有未批准偏差时，才能返回：

```text
状态：COMPLETED
Task ID：
Base commit：
产生的 commit：
修改文件：
Red 测试命令、退出码和关键失败：
Green 测试命令、退出码和结果：
完整模块验证：
git diff --check：
未执行的验证及原因：
与计划的偏差：
残余风险：
```

存在必选验证未执行、计划偏差未批准或工作区仍有当前 Task 未提交修改时，只能返回 `BLOCKED`，不得返回 `COMPLETED`。
