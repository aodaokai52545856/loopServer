**前置任务：** LE-P04-T02

## 唯一目标
跨 Linux/macOS/Windows 管理唯一 tag Runner 和并发配置

## 明确不做
- 不实现源计划当前 Task 以外的 Step、文件或后续 Task。
- 不改变 `01-不可变架构决策.md`。

## 前置条件
- 前置任务状态为 `ACCEPTED`；“无”时不检查前置 Task。
- 启动提示词中的分支、Base commit 和工作区状态与实际一致。
- 源计划当前 Task 列出的前置文件存在；不存在时返回 `BLOCKED`。

## 开始状态
- 当前分支和 HEAD 必须与动态启动提示词完全一致；detached HEAD 时返回 `BLOCKED`。
- `git status --short` 只能包含启动提示词逐项声明的已知偏差；出现其他路径时返回 `BLOCKED`。
- 运行源 Task 要求的基线或 Red 命令前，不得修改文件、安装依赖或创建 commit。

## 不可变约束
- 节点只读 clone 并生成候选 Artifact；OpenCode 环境不得含 GitLab/任务凭据；事件 append-before-send；验证命令不经过 shell。
- 业务状态变更和 Outbox 必须同事务；个人节点不得获得仓库写权限。

## 执行顺序
1. 完成只读预检并等待“开始执行 Task LE-P04-T03”。
2. 严格执行源 Task 的 Step 1，写失败测试。
3. 执行 Step 2，失败原因必须与计划一致。
4. 执行 Step 3 的最小实现，不提前实现后续 Task。
5. 执行 Step 4 的全部命令和本卡验收命令。
6. 运行 `git diff --check`，核对白名单和敏感信息。
7. 仅在启动提示词允许时执行 Step 5 commit；随后停止。
