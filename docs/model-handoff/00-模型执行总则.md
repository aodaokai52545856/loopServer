# 模型执行总则

## 1. 唯一任务
你每次只能执行启动提示词指定的一个 Task。完成或阻塞后立即停止，不得读取并开始下一 Task。

## 2. 强制读取顺序
依次读取本文件、`01-不可变架构决策.md`、当前任务卡、任务卡引用的计划章节、任务卡引用的架构章节、当前代码。不得用模型记忆替代工作区文件。

## 3. 两阶段协议
第一次只做只读预检并返回 `READY` 或 `BLOCKED`。只有收到“开始执行 Task <ID>”且 ID 与任务卡一致后才能修改文件。

## 4. 标准执行顺序
再次核对 HEAD/状态；编写失败测试；确认失败原因；最小实现；精确测试；模块测试；`git diff --check`；白名单/敏感信息检查；单 Task commit；完成报告。

## 5. 修改权限
只能修改任务卡白名单。不得 push、merge、rebase、reset、force、删除分支、amend、覆盖既有修改、改已提交 migration、升级未批准依赖或输出凭据。

## 6. 全局停止条件
脏工作区未声明、Base/分支不符、前置产物缺失、Red 失败原因不符、需要白名单外修改、文档与代码冲突、需要改变架构/状态/API/表/安全/版本、无法确认外部 API、必选验证不可运行、模块测试出现无关失败、同一阻塞三次无新证据时，必须返回 `BLOCKED`。

## 7. 完成门槛
必选验证全部执行且退出码为 0、当前 Task 已形成一个 commit、无未提交的当前 Task 修改、无未批准偏差，才可返回 `COMPLETED`。不得声称整个阶段完成。

## 8. 无 Superpowers
本协议不要求任何 Superpowers 技能。环境没有这些技能时按文档手工执行全部步骤，不得跳过。
