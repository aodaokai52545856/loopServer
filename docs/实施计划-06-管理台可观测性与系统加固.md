# 循环工程一期：管理台、可观测性与系统加固 Implementation Plan

> **执行说明：** 每次只执行一个 Task。若环境具备 `superpowers:subagent-driven-development` 或 `superpowers:executing-plans`，可以使用；不具备这些技能时，直接严格遵循本 Task 的 checkbox、Red→Green、验证和提交步骤。缺少技能不得成为跳步、伪造验证或扩大范围的理由。

**Goal:** 交付可供管理员、项目负责人、节点拥有者和观察者使用的管理台，并以权限、指标、告警、故障演练、样例项目和三平台发行验证完成一期上线门槛。

**Architecture:** 浏览器通过 GitLab OAuth2 登录；历史数据走分页 REST，实时事件走带断线续传的 SSE。Vue 按 feature 组织页面；控制面暴露 Prometheus 指标和结构化日志；端到端样例覆盖 Java、Vue2、Vue3，Linux/macOS 正式支持，Windows 保持 Beta 标识。

**Tech Stack:** Spring Security OAuth2 Client/Resource Server、Micrometer/Prometheus、Vue 3、TypeScript、Pinia、Vue Router、Vitest、Playwright、GitLab CI、Go cross-build。

- [ ] **Step 3: Implement OAuth2 login and explicit route rules**

Add `spring-boot-starter-oauth2-client` and `spring-boot-starter-security`. Configure authorization/token/user-info endpoints from environment and request scopes `read_user openid profile email`. Map GitLab group membership from a configured administrator group and project ownership from `project_profile_owner`; never trust a role supplied by the browser.

```java
@Bean
@Order(1)
SecurityFilterChain nodeApi(HttpSecurity http, DeviceCertificateFilter deviceFilter) throws Exception {
    return http.securityMatcher("/api/node/v1/**")
        .authorizeHttpRequests(auth -> auth
            .requestMatchers(HttpMethod.POST, "/api/node/v1/enroll").permitAll()
            .anyRequest().authenticated())
        .addFilterBefore(deviceFilter, AnonymousAuthenticationFilter.class)
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .csrf(AbstractHttpConfigurer::disable).build();
}

@Bean
@Order(2)
SecurityFilterChain internalHooks(HttpSecurity http) throws Exception {
    return http.securityMatcher("/internal/webhooks/**")
        .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .csrf(AbstractHttpConfigurer::disable).build();
}

@Bean
@Order(3)
SecurityFilterChain humanApi(HttpSecurity http) throws Exception {
    return http
        .securityMatcher("/api/**", "/oauth2/**", "/login/**")
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/session").authenticated()
            .requestMatchers("/api/admin/**").hasRole("ADMIN")
            .requestMatchers(HttpMethod.GET, "/api/**").hasAnyRole("ADMIN","PROJECT_OWNER","NODE_OWNER","OBSERVER")
            .anyRequest().authenticated())
        .oauth2Login(Customizer.withDefaults())
        .csrf(csrf -> csrf.csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse()))
        .build();
}
```

`DeviceCertificateFilter.shouldNotFilter` returns true only for `POST /api/node/v1/enroll`; that single endpoint is authenticated by the expiring one-time invite. No other node route can bypass device authentication.

Method-level `ProjectAuthorization` must still check project ownership for edits/retries and node ownership for pause/drain. Session cookie is `Secure`, `HttpOnly`, `SameSite=Lax`, eight-hour maximum. Redact OAuth access/refresh tokens from logs and never return them through `/api/session`.

- [ ] **Step 3: Implement safe forms and one-time secret display**

Node list/detail displays name, owner, description, OS/arch, tools, concurrency/active slots, CPU/memory/disk, heartbeat age, Runner state, project allowlist, desired/applied revision, last error and recent attempts. Invite code appears once after creation with expiry and `repair-node join` command; after the dialog closes only invite ID/status remain.

Operations pages expose Webhook/Outbox state, attempt count, next retry and redacted error. Retry and cancel actions require a reason of 10–500 characters and show the resulting audit ID.

Implement these authenticated mutation APIs; each requires a reason, performs object-level authorization and writes audit in the same transaction:

```text
POST  /api/admin/node-invites
POST  /api/nodes/{nodeId}/drain
POST  /api/nodes/{nodeId}/resume
POST  /api/nodes/{nodeId}/disable
PATCH /api/nodes/{nodeId}/configuration
POST  /api/nodes/{nodeId}/certificate-rotation
POST  /api/tasks/{taskId}/retry
POST  /api/tasks/{taskId}/cancel
POST  /api/operations/outbox/{eventId}/retry
POST  /api/operations/webhooks/{eventUuid}/retry
```

`V6__governance.sql` creates `node_certificate(node_id,serial,public_key_sha256,status,not_before,not_after,created_at,revoked_at)` and migrates the current certificate from `repair_node`. Rotation sets a desired action; the node generates a new key/CSR and calls `/api/node/v1/certificates/rotate` using its current certificate. The new certificate becomes `ACTIVE`, the old one `GRACE` for five minutes and then `REVOKED`. Disable revokes all certificates and deletes the Runner through GitLab after active Jobs finish or are explicitly cancelled.
