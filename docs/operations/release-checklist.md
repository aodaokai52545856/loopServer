# 发布与运维检查清单

在将 repair-node 包标为可分发、或推动控制面/Publisher 变更进入生产前，完成下列门禁。任一项未完成不得宣称正式发布。

## 节点包

- [ ] Linux amd64/arm64 与 macOS amd64/arm64 发布作业全部通过（阻塞）。
- [ ] Windows amd64 作业产物后缀为 `-beta`，且文档/SUPPORT 明确 Beta；失败时 `allow_failure` 不得被改写成 GA。
- [ ] 每个包含二进制、sample config、service 定义、LICENSE、SHA-256；已提交公司签名服务或按门禁记录延期原因。
- [ ] 安装器仅校验、复制、创建状态目录并打印 join；抽测确认无静默注册/下载 Runner/改防火墙。

## 凭据与证书

- [ ] 设备 CA 备份可用，并完成一次轮换演练（签发→节点重配→旧证吊销）。
- [ ] Publisher/GitLab 机器人 Token 完成轮换，范围仍为最小项目集；节点侧确认无写 Token。
- [ ] 邀请码明文仅管理台一次性展示；存储仅为哈希。

## 数据与恢复

- [ ] 数据库恢复演练：从备份恢复控制面 PostgreSQL，Outbox/状态可续跑且不重复创建分支/MR。
- [ ] Artifact 到期策略抽查：摘要与 MR/commit 链接仍可追溯。

## 稳定性

- [ ] 24 小时 soak：至少五路并发 Job，观察心跳、调度、OpenCode、发布与告警。
- [ ] 日志密钥扫描：无 Token、私钥、模型密钥明文。
- [ ] 回滚演练：控制面与节点包各演练一次回退路径并记录耗时。
